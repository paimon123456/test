# 库存管理与销售管理模块 - 功能说明

## 📋 本次新增内容总览

### 一、数据库变更（2张新表 + 测试数据）

#### 1. 新增表：`inventory_check`（库存盘点表）
**用途**：记录药品盘点业务流程，确保账实相符

**字段说明**：
- `check_id`: 盘点单ID（主键）
- `drug_id`: 药品ID
- `system_stock`: 系统库存数量
- `actual_stock`: 实际清点数量
- `diff_num`: 差异数量（自动计算）
- `diff_reason`: 差异原因
- `check_date`: 盘点日期
- `checker_id`: 盘点人ID
- `auditor_id`: 审核人ID
- `audit_status`: 审核状态（待审核/已通过/已驳回）
- `adjust_status`: 库存调整状态（未调整/已调整/无需调整）

#### 2. 新增表：`sales_return`（销售退货表）
**用途**：管理销售退货流程，支持退货审核和库存回补

**字段说明**：
- `return_id`: 退货单ID（主键）
- `original_order_id`: 原销售单号
- `drug_id`: 药品ID
- `batch_no`: 批号
- `return_num`: 退货数量
- `return_reason`: 退货原因
- `refund_amount`: 退款金额
- `status`: 退货状态（申请中/审核中/已完成/已驳回）
- `operator_id`: 操作人ID
- `auditor_id`: 审核人ID
- `create_time`: 创建时间
- `audit_time`: 审核时间

#### 3. 测试数据
- ✅ 3个测试会员（张三、李四、王五）
- ✅ 4个测试销售订单（包含不同支付方式）

### 三、运行后可用的新增功能

#### 🎯 库存管理模块（/inventory）

##### 1. **药品入库管理** ✅
**API**: `POST /inventory/purchase-in`

**功能**：
- 批量药品入库（采购入库）
- 自动验证药品是否存在
- 同一药品+批号自动累加库存
- 不同批号创建新库存记录
- 自动计算库存状态（正常/近效期/过期）
- 事务保证数据一致性

**使用示例**：
```json
POST /inventory/purchase-in
{
  "orderId": "PO001",
  "operatorId": "1",
  "items": [
    {
      "drugId": "1",
      "purchaseNum": 100,
      "purchasePrice": 8.50,
      "batchNo": "B20240501",
      "productionDate": "2024-05-01",
      "expiryDate": "2026-05-01",
      "warehouseId": "a001000000000000000000000000001",
      "location": "A区-A02-01"
    }
  ]
}
```

##### 2. **库存盘点管理** ✅（全新功能）
**APIs**:
- `POST /inventory/check/create` - 创建盘点单
- `POST /inventory/check/input-stock` - 录入实际库存
- `POST /inventory/check/audit` - 审核盘点单
- `GET /inventory/check/list` - 查询盘点单列表
- `GET /inventory/check/{checkId}` - 查询盘点单详情

**功能**：
- 支持全盘和抽盘两种模式
- 自动生成盘点单（记录系统库存快照）
- 录入实际库存后自动计算差异
- 审核后自动调整库存数据
- 完整的盘点流程管控（待审核→已通过→已调整）

**使用示例**：
```json
// 创建全盘盘点单
POST /inventory/check/create
{
  "checkType": "全盘",
  "checkerId": "1",
  "warehouseId": "a001000000000000000000000000001"
}

// 录入实际库存
POST /inventory/check/input-stock?checkId=chk001&actualStock=498

// 审核盘点单
POST /inventory/check/audit?checkId=chk001&auditorId=1&approved=true
```

##### 3. **库存查询与预警** ✅（已有功能增强）
**APIs**:
- `GET /inventory/list` - 分页查询库存
- `GET /inventory/near-expiry` - 查询近效期药品
- `GET /inventory/warehouse/{warehouseId}` - 按仓库查询

#### 🎯 销售管理模块（/sales）

##### 1. **前台销售开单** ✅（核心功能）
**API**: `POST /sales/order/create`

**功能**：
- ⭐ **先进先出、近效期先出**原则自动匹配库存
- 自动验证库存充足性
- 事务保证库存扣减和订单创建的原子性
- 自动计算总金额、优惠和实收金额
- 自动累计会员积分（每消费1元积1分）
- 支持多种支付方式（现金/微信/支付宝/医保/银行卡）
- 自动创建销售订单和销售明细

**使用示例**：
```json
POST /sales/order/create
{
  "memberId": "m001",
  "cashierId": "1",
  "payType": "微信",
  "discount": 5.00,
  "pointsUsed": 0,
  "items": [
    {
      "drugId": "1",
      "saleNum": 2,
      "salePrice": 12.00
    },
    {
      "drugId": "2",
      "saleNum": 1,
      "salePrice": 9.50
    }
  ]
}
```

##### 2. **销售订单查询与追溯** ✅
**APIs**:
- `GET /sales/order/list` - 查询订单列表（支持多条件筛选）
- `GET /sales/order/{orderId}` - 查询订单详情（包含明细）
- `GET /sales/order/stat/date` - 按日期统计销售数据
- `GET /sales/order/stat/drug` - 按药品统计销售数据
- `GET /sales/order/stat/total` - 统计销售总额

**功能**：
- 多维度查询（订单号、会员、收银员、状态、时间范围）
- 订单详情包含完整明细列表
- 销售数据统计（按日期、按药品、总额）
- 支持分页查询

**使用示例**：
```
GET /sales/order/list?startDate=2024-05-01&endDate=2024-05-10&pageNum=1&pageSize=10
GET /sales/order/so001
GET /sales/order/stat/date?startDate=2024-05-01&endDate=2024-05-10
```

##### 3. **销售退货管理** ✅（全新功能）
**APIs**:
- `POST /sales/return/create` - 创建退货申请
- `POST /sales/return/audit` - 审核退货申请
- `GET /sales/return/list` - 查询退货单列表
- `GET /sales/return/{returnId}` - 查询退货单详情

**功能**：
- 强制关联原销售订单
- 验证退货数量和批号匹配
- 审核通过后自动回补库存
- 自动更新原订单状态为"已退货"
- 完整的退货追溯链条

**使用示例**：

### 四、核心技术特性

#### 🔒 事务管理
- 所有涉及库存变动的操作都使用 `@Transactional` 保证数据一致性
- 销售开单：库存扣减 + 订单创建 + 明细创建 + 积分累计（原子操作）
- 退货审核：库存回补 + 订单状态更新（原子操作）
- 盘点审核：库存调整 + 盘点状态更新（原子操作）

#### 📊 智能库存匹配
- 销售时按**效期优先**原则自动匹配库存批次
- 自动遵循**先进先出**策略
- 库存不足时自动拒绝交易

#### 🎨 状态自动管理
- 库存状态根据有效期自动更新：
  - 过期 < 0天 → "过期"
  - 近效期 0-90天 → "近效期"
  - 正常 > 90天 → "正常"

#### 🔗 数据关联查询
- 所有查询都通过MyBatis XML实现关联查询
- 自动关联药品名称、用户姓名、会员信息等
- 减少前端多次请求

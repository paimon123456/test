# 库存管理与销售管理模块功能完善说明（无新增表方案）

## 一、方案概述

**核心原则：不新增数据库表，充分利用现有表结构**

### 实现策略

1. **库存变动日志** - 使用现有的 `sys_log` 表
2. **库存预警** - 实时查询 `drug_inventory` 表，不存储预警记录
3. **销售折扣优化** - 纯业务逻辑优化，无需新表

---

## 二、数据库使用

### 1. sys_log 表（已有）- 用于库存变动日志

**原用途**：系统操作日志  
**新用途**：同时记录库存变动日志

**使用方式：**
- `module` 字段：固定为 "库存管理"
- `type` 字段：变动类型（采购入库/销售出库/盘点调整等）
- `content` 字段：JSON格式存储详细信息
  ```json
  {
    "inventoryId": "xxx",
    "drugId": "xxx", 
    "batchNo": "xxx",
    "changeType": "销售出库",
    "changeNum": -5,
    "beforeStock": 100,
    "afterStock": 95,
    "relatedOrderId": "so001",
    "remark": "销售出库"
  }
  ```

### 2. drug_inventory 表（已有）- 用于库存预警

**新增功能**：实时查询低于最低库存的药品

**查询条件：**
```sql
SELECT * FROM drug_inventory 
WHERE min_stock > 0 
  AND stock_num < min_stock
```

## 四、功能详细说明

### 1. 库存变动日志记录

#### API接口
```
GET /inventory/log/list
参数：
- drugId (可选): 药品ID
- changeType (可选): 变动类型
- startDate (可选): 开始日期
- endDate (可选): 结束日期
- pageNum: 页码，默认1
- pageSize: 每页数量，默认10
```

#### 记录时机
- **销售出库**：在 `SalesOrderServiceImpl.createSalesOrder()` 中
- **采购入库**：在 `PurchaseInServiceImpl.purchaseIn()` 中

#### 日志内容示例
```json
{
  "inventoryId": "inv001",
  "drugId": "1",
  "batchNo": "B20240101",
  "changeType": "销售出库",
  "changeNum": -5,
  "beforeStock": 500,
  "afterStock": 495,
  "relatedOrderId": "so001",
  "remark": "销售出库"
}
```

### 2. 库存预警功能

#### API接口

**查询低库存药品列表：**
```
GET /inventory/alert/low-stock
参数：
- pageNum: 页码，默认1
- pageSize: 每页数量，默认10
```

**获取低库存数量：**
```
GET /inventory/alert/low-stock/count
```

#### 预警规则
- 条件1：`min_stock > 0`（设置了最低库存阈值）
- 条件2：`stock_num < min_stock`（当前库存低于阈值）

#### 特点
- ✅ 实时查询，数据准确
- ✅ 无需定时任务
- ✅ 无需额外存储空间
- ✅ 自动反映最新库存状态

### 3. 销售折扣处理优化

#### 支持的优惠方式
1. **固定金额优惠** - `discount` 字段
2. **会员积分抵扣** - `pointsUsed` 字段（100积分 = 1元）

#### 验证规则
- 优惠金额不能超过订单总额
- 会员积分必须足够
- 总优惠（优惠金额 + 积分抵扣）≤ 订单总额
- 实收金额 ≥ 0

#### 积分规则
- **获取积分**：按实收金额计算，1元 = 1积分
- **使用积分**：100积分 = 1元
- **流程**：先扣除使用的积分 → 计算实收金额 → 按实收金额增加积分

## 七、使用示例

### 1. 查询库存变动日志

```bash
# 查询某药品的所有变动记录
GET /inventory/log/list?drugId=1&pageNum=1&pageSize=20

# 查询某时间段内的销售出库记录
GET /inventory/log/list?changeType=销售出库&startDate=2024-01-01&endDate=2024-01-31
```

### 2. 查询库存预警

```bash
# 查询所有低库存药品
GET /inventory/alert/low-stock?pageNum=1&pageSize=50

# 获取低库存药品数量
GET /inventory/alert/low-stock/count
```

### 3. 销售开单（带折扣）

```json
POST /sales/order/create
{
  "memberId": "m001",
  "cashierId": "1",
  "payType": "微信",
  "discount": 5.00,
  "pointsUsed": 200,
  "items": [
    {
      "drugId": "1",
      "saleNum": 2,
      "salePrice": 12.00
    }
  ]
}
```

**计算过程：**
- 订单总额：2 × 12.00 = 24.00元
- 优惠金额：5.00元
- 积分抵扣：200 ÷ 100 = 2.00元
- 实收金额：24.00 - 5.00 - 2.00 = 17.00元
- 获得积分：17分

## 八、注意事项

### 1. sys_log表的使用
- 库存日志与其他系统日志共用一张表
- 通过 `module = "库存管理"` 区分
- 建议定期清理旧日志（如保留6个月）

### 2. 库存预警
- 预警是实时查询，每次请求都会扫描数据库
- 如果数据量大，建议在 `drug_inventory` 表的 `min_stock` 和 `stock_num` 字段上建立索引
- 已存在索引：`KEY idx_drug (drug_id)`，可以考虑添加复合索引

### 3. 性能优化建议
```sql
-- 可选：为库存预警查询添加索引
ALTER TABLE drug_inventory ADD INDEX idx_stock_alert (min_stock, stock_num);
```

### 4. 数据一致性
- 库存变动和日志记录在同一事务中
- 确保日志记录失败不影响主业务流程
